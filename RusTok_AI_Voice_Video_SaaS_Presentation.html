⚠️ Проблемы существующего рынка
  • Потеря контекста между сессиями: Обычные TTS сервисы генерируют каждую фразу с нуля, меняя тембр и произношение.
  • Сложности с хронометражем при дубляже: Переведенный текст на 20-30% длиннее оригинала, ломая синхронизацию с видео.
  • Высокая стоимость и фрагментация: Отдельные сервисы для текста, аудио, перевода и хранения.
💡 Наше Решение на RusTok
  • Непрерывная RAG-память: Единая "библия проекта" и словарь правил гарантируют стабильность голоса и терминов.
  • Умная адаптация длины фраз: AI-оркестратор подгоняет перевод под точные таймкоды исходника.
  • Единая экосистема: Контекст + Озвучка + Видео + Биллинг в одной модульной Rust-системе.

03 / ИИ И КОНТЕКСТ

Трехуровневая Система Памяти Контекста

Как система обеспечивает идеальную точность тональности и произношения
1. Сессионный контекст
Short-Term Session Memory
  • История взаимодействия в ai_chat_sessions.
  • Сохранение оперативных правок ("сделай паузу", "озвучь более эмоционально").
  • Динамическое управление окном токенов.
2. Долгосрочный RAG
Long-Term Knowledge Memory
  • Векторное хранилище в PostgreSQL (Athanor).
  • "Библия проекта", характеры персонажей, сюжетный контекст предшествующих глав.
  • Автоматический контекстный поиск перед синтезом.
3. Голосовой профиль
Speaker & Lexicon Profile
  • Пользовательские словари ударений и исключений (IPA/Lexicon).
  • Эмбеддинги клонированного голоса.
  • Стилевые промпты тембра и эмоциональных выражений.
Результат: Полное отсутствие "сброса памяти" диктора при озвучке длинных серий видео или многотомных аудиокниг.

05 / РАЗРАБОТКА

Новый Специализированный Модуль rustok-voice

Целевое расширение экосистемы RusTok для работы со звуком
🎙️ Функциональные Обязанности Модуля
  • TTS & STT Abstraction Layer: Единый trait-интерфейс для ElevenLabs, Cartesia, PlayHT и локальных Kokoro/CosyVoice моделей.
  • Phoneme & Lexicon Engine: Обработка фонетических правил, ударений и SSML-разметки пауз.
  • Audio Processing Pipeline: Сведение дикторского голоса с фоновой музыкой и эффектами.
  • Voice Cloning Management: Управление емкостью голосовых слепков арендаторов.
⚙️ Технический Стек
  • Backend: Rust (Tokio, Tonic gRPC, Axum) для сверхбыстрой обработки потоков.
  • AI Runner: Rig 0.39 + ONNX Runtime / PyTorch sidecars через gRPC.
  • Media Engine: FFmpeg bindings + Rust native audio decoders.
  • Frontend Packages: Leptos SSR/Wasm & Next.js UI пакеты в концепции FFA (Fluid Frontend).
